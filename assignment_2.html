Heart Disease Dataset

This dataset contains data on heart disease diagnosis. It contains 14 columns, including one column (the column target) that indicates the presence (or absence) of heart disease in the patient. Missing values in this dataset are specified with ? in the data files.

This dataset was created by the following institutions and principal investigators:

  1. Hungarian Institute of Cardiology. Budapest: Andras Janosi, M.D.
  2. University Hospital, Zurich, Switzerland: William Steinbrunn, M.D.
  3. University Hospital, Basel, Switzerland: Matthias Pfisterer, M.D.
  4. V.A. Medical Center, Long Beach and Cleveland Clinic Foundation: Robert Detrano, M.D., Ph.D.

For more information on this dataset, see https://archive.ics.uci.edu/ml/datasets/heart+disease.

Parkinson Speech Dataset

This dataset was created by Sakar et al. [1] and contains data from speech recordings of subjects with Parkinson’s disease as well as healthy subjects. It contains 31 columns, including one column (the column class) that indicates the presence (or absence) of Parkinson’s disease and columns containing time-frequency-based features extracted from voice samples. Missing values in this dataset are specified with n/a in the data files.

For more information on this dataset, see Sakar et al. [1] and
https://archive.ics.uci.edu/ml/datasets/Parkinson+Speech+Dataset+with++Multiple+Types+of+Sound+Recordings.

Class Dataset (2P)

For this assignment, you will extend the functionality of the Dataset class that you wrote in Assignment 1. As in Assignment 1, the class Dataset holds the data and provides dataset-related functionality. The extended Dataset class must have all the properties and methods that were required in Assignment 1. Additionally, the extended version of the Dataset class must have the following functionality:

Required Additional Method: split_data

  • Functionality:
    This method splits the dataset into a training and a test set. The training set will later be used to train a classifier; the test set will be used to evaluate the predictive performance of a classifier. You can use the method train_test_split [3] from sklearn.model_selection in your implementation of split_data.

    The method split_data uses the following information provided in the config file:

    • random_state_for_split: The random state for the train/test split. Setting a random state allows for reproducible output (see [3] for further details).
    • test_size: The size of the test set. This should be a number between 0.0 and 1.0, representing the fraction of the data that you want to include in your test set (see [3] for further details).
    • features_to_use_for_classification: A list of the names of the features that you want to use for training the classifier. The value "all" indicates that all features (except the target variable) should be used for classification.

    Hint: It is probably a good idea to extend the Dataset's __init__ method so that it stores these three specified values in non-public instance attributes.

    The method split_data also provides the option to impute missing values according to the strategy specified in the parameter impute_strategy. You can use sklearn.impute.SimpleImputer [4] to do this. The following imputation strategies (i.e., allowed values for the parameter impute_strategy) should be supported:

    • "mean": This strategy replaces any missing values of a feature by the mean of that feature in the training set (also see [4]).
    • "median": This strategy replaces any missing values of a feature by the median of that feature in the training set (also see [4]).
    • "most_frequent": This strategy replaces any missing values of a feature by the most frequently occurring value of that feature in the training set (also see [4]).
    • None: Completely removes all rows where any feature has a missing value.

    [3] https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.train_test_split.html
    [4] https://scikit-learn.org/stable/modules/generated/sklearn.impute.SimpleImputer.html

  • Parameters:

    Name Type Default Value Description
    impute_strategy str None The imputation strategy for filling in missing values (see functionality description for details).

  • Return:

    Type Shape Description
    numpy.ndarray (<num_train_samples>, <num_features>) The feature matrix of the training set, i.e., the values of all features to be used in the classification as specified in features_to_use_for_classification in the config file
    numpy.ndarray (<num_train_samples>,) The target vector (i.e., the values of the target variable) of the training set.
    numpy.ndarray (<num_test_samples>, <num_features>) The feature matrix of the test set, i.e., the values of all features to be used in the classification as specified in features_to_use_for_classification in the config file
    numpy.ndarray (<num_test_samples>,) The target vector (i.e., the values of the target variable) of the test set.

Class EvaluationMetrics (5P)

The class EvaluationMetrics is responsible for calculating various evaluation metrics. Only use numpy and the standard library in your implementation of this class. It is sufficient if your implementation is able to handle binary target variables (i.e., you don’t have to consider cases with more than two possible target values).
Note: The term “class” in the description of the properties and the __str__ method refers to the target variable and not to the object-oriented programming concept of classes. <num_classes> is the number of different target variables (2 in the binary case).

Required Properties

Name Type Shape Getter Setter Description
confusion_matrix numpy.ndarray (<num_classes>,<num_classes>) yes no A confusion matrix in the same format as in [5].
FP numpy.ndarray (<num_classes>,) yes no The number of false positives for each class.
FN numpy.ndarray (<num_classes>,) yes no The number of false negatives for each class.
TP numpy.ndarray (<num_classes>,) yes no The number of true positives for each class.
TN numpy.ndarray (<num_classes>,) yes no The number of true negatives for each class.
precision numpy.ndarray (<num_classes>,) yes no The precision for each class.
recall numpy.ndarray (<num_classes>,) yes no The recall for each class.
F1 numpy.ndarray (<num_classes>,) yes no The F1-score for each class.
accuracy_score numpy.float64 yes no The accuracy of the classifier.

For all properties listed above, the entry for a specific class label (i.e. unique target variable) can be found by using it as an index. E.g., FP[0] gives the number of false positives for class 0, TN[1] gives the number of true negatives for class 1.

[5] https://scikit-learn.org/stable/modules/generated/sklearn.metrics.confusion_matrix.html

Required Method: __init__

  • Functionality:
    The values of the parameters y_true and y_pred are stored in non-public instance attributes.

  • Parameters:

    Name Type Shape Description
    y_true numpy.ndarray (<num_test_samples>,) The true values of the target variable (obtained from the test set)
    y_pred numpy.ndarray (<num_test_samples>,) The values of the target variable that a classifier predicted

Required Method: __str__

Required Method: __init__

    dataset Dataset The dataset you want to use for your classification tasks.

Required Method: best_classifier

  • Functionality:
    This method receives a list of classifier objects and determines which classifier has the best accuracy. The method trains (i.e. fits) each classifier in the list on the training set and uses each trained classifier to predict the target values for the test set. The method also creates an EvaluationMetrics object for each classifier and stores them in an instance attribute.
    The method returns the classifier object that reached the highest accuracy.

  • Parameters:

    Name Type Description
    classifiers list A list of classifier objects to be compared. Each classifier in the list must have the methods fit and predict implemented.

  • Return:

Class kNN (5P)

The class kNN implements the k-nearest neighbors (KNN) classification algorithm. It uses cosine distance (i.e., 1 - cosine similarity) as the distance metric. Only use numpy and the standard library in your implementation of this class.
Note: You will learn about this algorithm in detail in the tutorials. There is also a basic explanation here and a more thorough and mathematical one on Wikipedia.
Note: You can use sklearn.neighbors.KNeighborsClassifier (with the parameter metric='cosine') to check whether your implementation works correctly (but do not use it in your implementation ;).

Required Method: __init__

  • Functionality:
    The value of the parameter n_neighbors is stored in a public instance attribute of identical name.

  • Parameters:

    Name Type Description
    n_neighbors int How many neighbors should be taken into account

Required Method: fit

  • Functionality:
    This method stores the parameters in non-public instance attributes.
    Note: The fit method of a classifier usually learns from the training data. Since the k-nearest neighbors classifier works by comparing every test sample to every training sample, this usual training step is not applicable. It is however possible to do part of the calculation of the cosine distance using only the training data. You can decide how to handle this implementation detail yourself.

  • Parameters:

    Name Type Shape Description
    X_train numpy.ndarray (<num_train_samples>,<num_features>) The feature matrix of the training set.
    y_train numpy.ndarray (<num_train_samples>,) The target vector of the training set.

Required Method: predict

  • Functionality:
    This method predicts the values of the target variable for the provided feature matrix of the test set (parameter X_test). It uses cosine distance (i.e., 1 - cosine similarity) as the distance metric.
    Hint: Due to the small size of the used datasets the classification process will finish rather quickly, independently of the efficiency of your implementation. It is however always advisable to think about (1) which parts of the calculations are constant for every test sample and only have to be calculated once and (2) which calculations can be performed using array operations instead of loops.

  • Parameters:

    Name Type Shape Description
    X_test numpy.ndarray (<num_test_samples>,<num_features>) The feature matrix of the test set.

  • Return:

    Type Shape Description
    numpy.ndarray (<num_test_samples>,) The predicted values of the target variable.
